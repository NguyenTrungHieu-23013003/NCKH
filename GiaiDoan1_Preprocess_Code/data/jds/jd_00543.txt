The role of a Data Quality Analyst is to ensure the timely receipt and processing of client data in order to support company product lines and related reporting requirements. This role includes managing the file intake process and performing various data validations, mappings and audits to enforce defined data quality standards across all file types. The analyst works closely with both business users and other IT members to achieve these standards. To the extent that process and protocol need to be adjusted, the Data Quality Analyst is responsible for proposing concerns to the IT Operations Lead. File Intake Prioritize incoming DQT data requests to ensure timely handling. Validate and map client data files. Organize and store files on network file share. Work with Developers to load client data. Recommend process improvements to enhance traceability or streamline total intake time. Data Quality Review post load audits to identify, compare, and resolve data quality problems. Research and determine scope and complexity of data quality issue to identify steps to fix issue. Work with Developers to correct data quality errors. Recommend process improvements to enhance overall data quality. Ensure adherence to data quality standards. Resolve all internal data exceptions in timely and accurate manner. Analyze, query and manipulate data according to defined business rules and procedures. Email Marketing Process Client Service requests to create custom email templates in email marketing.
