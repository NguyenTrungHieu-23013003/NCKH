Vehicle Data System Engineers create the on-vehicle enablers that allow flexible back office access to vehicle data enabling new features and data analysis. They develop the concepts and requirements of vehicle data collection mechanisms for our next generation vehicles. To that end, they improve the vehicle data architecture, develop interfaces to the back office, and comprehend the needed interactions between the vehicle data system and the back office driven by our growing data collection needs. They work with and lead cross-domain teams to develop and extend data collection capabilities. While the role is broad, there is a focus on data and transport security, securing the on-vehicle data processing and working security and privacy policy requirements. Key Security and Policy responsibilities include: Develop next generation data elements handling systems including on-vehicle and back office policy management of each data element Develop new strategies for dynamic vehicle data processing to share on-vehicle resources with normal vehicle functions Align vehicle data systems with legal and policy requirements globally Develop security for the next generation data processing system (hypervisor, sandboxing, etc.) Develop strategy and requirements for data transmission from the vehicle aligned with security requirements Further vehicle data responsibilities may include: Develop next generation vehicle to back office communication protocols Develop next generation on-vehicle data processing architecture Supporting API development for vehicle data interfaces and backoffice communication Coordinate with the vehicle data future roadmap strategy supporting exponentially growing data needs and data source access The following are expected outputs for this System Engineer role: Lead Cross-Functional, end-to-end development teams working thru each data element storage, transport and use to further Big Data goals.
